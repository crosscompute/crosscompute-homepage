Custom Service: Data Warehouse Integration

Problem
  • Multiple data sources make report preparation time-consuming.
Proposed Solution
  • Provide a single access point for datasets by populating a Data Warehouse with version control. Code for the Data Warehouse will be owned by CrossCompute and will be open source under an MIT License. The Data Warehouse will only be accessible programmatically via its REST API and will not have a web interface.
  • Automate dataset updates by building data pipelines that aggregate internal and external data sources into the Data Warehouse. Code for the data pipelines to update the Data Warehouse will be owned by your organization.
Expected Benefit
  • Simplify analysis for commonly used datasets.
  • Serve visualizations and reports even when data sources are temporarily down.
  • Track changes to spatial and tabular data.
Scope of Work
Phase: Report Dataset
  1. Identify a dataset needed to generate a specific report.
  2. Write scripts to update the dataset from internal and external data sources.
  3. Configure a REST server to serve the dataset and authenticate requests using public-key cryptography.
Cost
4-7 Support Tickets per Phase depending on dataset complexity

Custom Service: Interactive Information Filters

Problem
  • Readers cannot interactively filter data in visualizations in your report.
  • Filters are search queries that pinpoint information by hiding irrelevant data.
Proposed Solution
  • Let users filter data in your visualizations. Code for the interactive view plugins will be owned by CrossCompute and will be open source under an MIT License.
  • Check that users can interact with information filters for widgets embedded in reports and websites.
Expected Benefit
  • Help report readers and website visitors pinpoint information.
Scope of Work
Phase: Custom Widget with Filters
  1. Make an example widget with data for testing data filter functionality.
  2. Implement custom filters for widget.
Cost
3-5 Support Tickets per Phase depending on widget complexity

Custom Service: Hazard Mitigation Grant Proposal Wizard

Problem
  • Gathering data to prepare a Hazard Mitigation Grant Proposal is hard.
Proposed Solution
  • Deploy a web-based GIS to help your community gather data to prioritize infrastructure upgrades. Code for Infrastructure Planner will be owned by CrossCompute and will be open source under an MIT License.
  • Draft documentation so that your community can deploy Infrastructure Planner securely on your own servers.
  • Deploy a Hazard Mitigation Grant Proposal Wizard. Code for the Hazard Mitigation Grant Proposal Wizard will be owned by CrossCompute and will be open source under an MIT License.
Expected Benefit
  • Help your community win Hazard Mitigation Grants to upgrade infrastructure.
Scope of Work
Phase: Benefit Cost Analysis
  1. Identify recommended hazard mitigation strategies for your community.
  2. Identify data needed to justify each hazard mitigation strategy for benefit cost analysis (see Benefit Cost Analysis).
  3. Draft a wizard that your community can use to prepare a Hazard Mitigation Grant Proposal.
Phase: GIS Data
  1. Find external sources that provide GIS data for each hazard mitigation strategy.
  2. Customize Infrastructure Planner so that your community can gather the remaining GIS data.
  3. Write tutorials to help your community install and use Infrastructure Planner and the Hazard Mitigation Grant Proposal Wizard.
Phase: Cost Data
  1. Find external sources that provide cost data for each hazard mitigation strategy.
  2. Customize Infrastructure Planner so that your community can gather the remaining cost data.
  3. Write tutorials to help your community install and use Infrastructure Planner and the Hazard Mitigation Grant Proposal Wizard.
Cost
10 Support Tickets per Phase
